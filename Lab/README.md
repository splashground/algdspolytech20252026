## Вариант 20: Очередь с приоритетами
* Реализуйте очередь с приоритетами с помощью связного списка
* Напишите тесты для проверки корректности работы
* Сравните очередь с приоритетами и обычную очередь, укажите преимущества и недостатки

---

## Очередь с приоритетами на основе связного списка
### Преимущества
* Доступна динамическая расширяемость — нет ограничения на размер очереди
* Элементы вставляются с учётом приоритета, обеспечивая корректную обработку критичных задач
* Нет необходимости заранее резервировать память, как в массиве
* Гибкость: можно легко изменять структуру и добавлять дополнительные поля

### Недостатки
* Отсутствует локальность данных — элементы разбросаны в памяти
* Дополнительные расходы на хранение указателей и накладные расходы на выделение памяти
* Сложность реализации — требуется аккуратно управлять связями между узлами
* Немного более низкая производительность по сравнению с массивной реализацией из-за динамического выделения памяти

## Сравнение с обычной очередью на основе массива
### Преимущества массива
* Быстрый доступ к элементам по индексу
* Меньшие накладные расходы на память — нет указателей
* Простая реализация для фиксированного размера очереди

### Недостатки массива
* Ограниченный размер, требует заранее выделенной памяти
* Сложнее вставлять элементы по приоритету — требуется сдвиг элементов
* Меньшая гибкость и расширяемость
